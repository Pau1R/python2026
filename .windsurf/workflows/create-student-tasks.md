Твоя задача. сгенерировать индивидуальные практические задания для **всех студентов** по одному выбранному уроку. Задания должны быть привязаны к **блокам урока** и уровню каждого студента.

**Важно**.
- Воркфлоу не должен объединять студентов или темы логически.
- Для каждого студента и каждого блока задания формируются **отдельно**, без общих шаблонов.
- Даже если нужно сделать 44 задач. ты создаёшь их **по очереди**, по одному заданию за раз. строго следуя шагам.
- После первоначальных вопросов к пользователю процесс генерации идёт **без остановок**: ИИ не задаёт новых вопросов, все ответы ищет только в уже существующих файлах.

## 1. Входные данные и ограничения

1. Запроси у пользователя:
   - номер урока `<lesson_number>`
   - дату урока `<lesson_date>`
   - точное название урока `<lesson_title>` (как в структурах `lessons`)
2. Используй файлы:
   - `python2026/course.md`
     - для проверки. что урок существует и тема совпадает
   - `python2026/lessons/Урок <lesson_number> (<lesson_date>). <lesson_title>.md`
     - подробное содержание урока для студентов
     - в этом файле обязательно должны быть 3 или 4 блока темы
   - `python2026_tasks/lessons/lesson <lesson_number> (<lesson_date>). <lesson_title>/notes.md`
     - конспект преподавателя. цель. тезисы и план
   - `python2026_tasks/students/all_performance.md`
     - общая сводка по уровням и типичным ошибкам студентов
3. Для **каждого** студента:
   - `python2026_tasks/students/<студент>/performance.md`
     - индивидуальный уровень и ошибки

Если какой‑то обязательный файл отсутствует. остановись и сообщи. что именно не найдено.

## 2. Определение структуры урока и блоков

1. По файлу содержания урока в репозитории преподавателя:
   - определи количество блоков темы: 3 или 4
   - для каждого блока запиши:
     - название блока
     - краткое описание
     - ключевые понятия блока
2. Свяжи каждый блок с планом из `notes.md`:
   - на какой части занятия этот блок объясняется
   - есть ли в плане явная практика по этому блоку

## 3. Список студентов и их уровень

1. Построй список студентов по директории:
   - `python2026_tasks/students/<студент>/performance.md`
2. Для **каждого** студента:
   - проанализируй `performance.md` и `all_performance.md`

     - слабый
     - средний

   - если уровень не указан считай за средний
   - выдели:
     - какие блоки урока вызывают больше всего потенциальных проблем
     - какие навыки стоит усилить (перечень ≤ 5 пунктов)

## 4. Общие правила генерации заданий

1. Все студенты **должны получить задания по каждому блоку** урока.
2. Для каждого студента и блока генерируй **отдельное задание**:
   - нельзя использовать общий шаблон и просто подставлять имя или числа
   - контекст. данные. сюжет. входные/выходные значения должны отличаться
3. Уровень сложности:
   - студент без опыта программирования
   - пиши простым языком. объясняй термины
   - избегай громоздких формулировок и профессионального жаргона
4. Время:
   - на блок теории в занятии ~15 минут
   - на каждое практическое задание 15–20 минут выполнения
   - подбирай объем и сложность так. чтобы задание было реалистично выполнимым за это время
5. Соблюдай ограничение:
   - в каждом списке не более 5 пунктов на одном уровне иерархии

## 5. Места и формат записи файлов

Для каждого студента `<student>` и каждого блока `<block_number>` урока:

1. Базовый путь урока в задачах:
   - `python2026_tasks/lessons/lesson <lesson_number> (<lesson_date>). <lesson_title>/tasks/`
2. Для каждого блока:
   - создай (если нет) подпапку
     - `tasks/<block_number>/`
3. Для каждого студента и блока создай **отдельный файл**:
   - имя файла:
     - `task <lesson_number>.<block_number> <student>.md`
   - путь:
     - `python2026_tasks/lessons/lesson <lesson_number> (<lesson_date>). <lesson_title>/tasks/<block_number>/task <lesson_number>.<block_number> <student>.md`
4. Если файл уже существует:
   - **не перезаписывай**
   - можешь только дополнять или аккуратно исправлять по явным указаниям воркфлоу. без удаления существующих условий задач

## 6. Структура каждого задания (файл `task ... .md`)

Структура одинакова по разделам. но содержание **строго индивидуально**:

1. `# Задание <lesson_number>.<block_number>. <lesson_title>`
2. `## Блок урока`
   - название и краткое пояснение. чем этот блок важен
3. `## Условие задачи`
   - чётко и просто сформулированная задача
   - без лишнего текста. только необходимый контекст
4. `## Пример входных данных`
   - 1–3 примера (не более 3)
5. `## Пример выходных данных`
   - ожидаемый результат для приведённых входных данных
6. `## Критерии успешного выполнения`
   - 3–6 пунктов. что считается правильно выполненным решением
7. `## Подсказки`
   - если уровень слабый. 3–6 пошаговых подсказок от общих к более конкретным
   - если средний. 2–4 подсказки среднего уровня абстракции
   - если сильный. 1–2 общих направления без готового решения
8. `## Частые ошибки и ограничения`
   - 2–5 типичных ошибок для этого блока темы
   - ограничения по входным данным. граничные случаи. что нельзя использовать

Все списки в этом файле должны иметь не более 5 элементов на одном уровне.

## 7. Последовательность генерации (важно для 44+ задач)

1. Обработай урок последовательно **по блокам**:
   1. блок 1
   2. блок 2
   3. блок 3
   4. блок 4 (если есть)
2. Для **каждого блока**:
   - возьми список всех студентов
   - для каждого студента:
     - определи уровень и проблемные навыки
     - сгенерируй **ровно одно** задание для этого студента и этого блока
     - создай или дополни соответствующий файл `task <lesson_number>.<block_number> <student>.md`
3. Не объединяй студентов в одну задачу и не создавай общие условия «для группы».
4. Не объединяй разные темы:
   - каждое задание жёстко привязано только к одному блоку текущего урока
5. Даже если суммарно получается 44 и более заданий:
   - ты создаёшь их **по одному за раз**, в порядке:
     - блоки по возрастанию номера
     - внутри блока студенты в алфавитном порядке по имени папки
