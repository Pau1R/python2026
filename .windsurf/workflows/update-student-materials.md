Этот воркфлоу запускается сразу после завершения `/create-student-tasks`. Цель. собрать все технические требования из сгенерированных заданий и убедиться. что они отражены в описании урока преподавателя. Если какого-то технического момента нет в lesson-файле. добавь его вручную.

## 1. Подготовка данных

1. Запроси у пользователя или найди в предыдущем шаге параметры урока:
   - `<lesson_number>`
   - `<lesson_date>`
   - `<lesson_title>`
2. Убедись. что существуют каталоги и файлы:
   - `python2026_tasks/lessons/lesson <lesson_number> (<lesson_date>). <lesson_title>/tasks/`
   - `python2026/lessons/Урок <lesson_number> (<lesson_date>). <lesson_title>.md`
3. Если чего-то нет. остановись и сообщи о недостающих данных.

## 2. Сбор технических моментов из заданий

1. Обойди все файлы `task <lesson_number>.<block_number> <student>.md` во всех подпапках `tasks/<block_number>/`.
2. Для каждого файла выпиши технические моменты. которые встречаются в разделах "## Условие задачи", "## Критерии успешного выполнения" и "## Частые ошибки и ограничения". Это могут быть конструкции языка, структуры данных, ограничения, алгоритмы.
3. Нормализуй формулировки:
   - переведи в именительный падеж
   - убери персонализации (имена студентов)
   - объединяй дубли (одинаковое требование из разных заданий)
4. Сформируй итоговый список (≤5 пунктов на уровень вложенности). разбив по блокам урока, чтобы сохранить связь с темами.

## 3. Сверка с файлом урока

1. Открой `python2026/lessons/Урок <lesson_number> (<lesson_date>). <lesson_title>.md`.
2. Найди существующий раздел про технические моменты. Если его нет, добавь раздел `## Технические моменты из практики` после описания всех блоков.
3. Для каждого элемента из итогового списка выполни поиск (регистронезависимый) в lesson-файле.
4. Пометь элементы, которых нет в файле. Они потребуют добавления.

## 4. Добавление недостающих пунктов

1. Сгруппируй отсутствующие пункты по блокам урока.
2. В выбранном разделе добавь подзаголовок `### Блок <n>. <название>` и маркированный список технических моментов блока.
3. Следи, чтобы в каждом списке было не более 5 элементов. Если больше. разбей по смысловым подпунктам.
4. Используй `apply_patch` для редактирования файла. соблюдай существующий стиль и язык урока.

## 5. Валидация

1. Снова прочитай lesson-файл и убедись, что каждый технический момент из заданий теперь упоминается хотя бы один раз.
2. Проверь, что структура раздела сохранена и не дублирует уже существующие пункты.
3. Сообщи пользователю, какие пункты добавлены и в какие разделы.
