## 4. Принципы индивидуализации домашних заданий

1. Домашнее задание каждого студента должно:
   - опираться на результаты **его** практических заданий из папки `tasks/` текущего урока
   - содержать задания по всем ключевым блокам урока
   - минимум 50% заданий должны быть связаны с теми местами. где студент допускал ошибки

2. Запрещено использовать общий шаблон:
   - нельзя копировать одну и ту же формулировку задания между студентами
   - нельзя ограничиваться заменой только чисел или имен. нужно менять контекст. сценарий. детали условий
   - структура разделов файла может быть одинаковой. но содержание задач. формулировки. примеры и ограничения должны отличаться
   - если несколько студентов имеют одинаковые проблемы. задания по идее могут быть похожими. но должны отличаться по:
     - сюжету
     - наборам входных данных
     - вариациям требований и ограничений

3. Учитывай уровень студента:
   - слабый. больше разбиения на подшаги, больше подсказок и явных критериев
   - средний. умеренное количество подсказок, акцент на отработку ошибок
   - сильный. меньше подсказок, больше открытых задач и расширений
   - если уровень не определён явно. используй настройки для **среднего** уровня

## 5. Структура файлов домашнего задания

Домашнее задание каждого студента должно быть представлено **в двух местах**.

### 5.1. В репозитории задач

1. Создай (если нет) папку:

   - имя файла должно однозначно содержать номер урока и имя студента

4. В конце файла обязательно укажи что анализируется только **первый** коммит, попавший в папку `homework/`. Если нужно добавить несколько файлов, загружать их одним коммитом.

### 5.2. В индивидуальном репозитории студента

1. Определи имя папки урока:
   - `<lesson_folder>` должно точно совпадать с именем папки урока в репозитории преподавателя:

2. Путь к файлу студента:

3. Если нужные папки не существуют:
   - создай `lessons/<lesson_folder>`
4. Если файл `homework <lesson_number>.md` не существует:
   - создай его и заполни содержимым на основе эталонной версии из репозитория задач
5. Если файл уже существует:
   - аккуратно обнови раздел для текущего урока
   - не удаляй данные о других уроках. если они присутствуют

## 6. Структура содержимого домашнего задания

В обоих файлах (в `python2026_tasks` и в репозитории студента) содержимое для текущего урока должно быть одинаковым по смыслу и структуре.

Рекомендуемая структура:

```markdown
# Домашнее задание. Урок <lesson_number>. <lesson_title>

## Повторение сложных тем
- 2–5 заданий
- каждое задание явно связано с конкретным блоком урока и конкретной ошибкой студента

## Закрепление ключевых тем
- 2–5 заданий
- покрывают все 3 или 4 блока урока
- допускается использование новых контекстов и примеров

## Подготовка к следующему уроку
- 1–3 задания
- задания подводят к следующей теме курса

## Дедлайны и критерии оценки
- четко указанные сроки сдачи
- прозрачные критерии оценки для всех типов задач
```

Во всех списках не должно быть более 5 элементов на одном уровне перечисления.

Для **каждого задания**:
- укажи:
  - формулировку задачи
  - связь с блоком урока
  - ожидаемый формат ответа
- добавь 2–4 либо больше конкретных критериев проверки
- при необходимости укажи ограничения:
  - форматы ввода
  - граничные случаи
  - запреты на использование нерелевантных библиотек
