## 5. Структура теоретического файла для студентов

Весь теоретический файл должен быть самодостаточным.
Не добавляй домашние задания и практические задания в этом файле. он только для теории.

5.1. Заголовок.
- первая строка файла.
  - `# Урок <номер>. <Название>`

5.2. Цель урока.
- раздел `## Цель урока`
- простое описание. чему студент научится к концу урока.
- используй 2-4 коротких предложения.

5.3. Ключевые понятия.
- раздел `## Ключевые понятия`
- построй иерархический список понятий по содержанию темы.

  - если понятий много. группируй их во вложенные списки
- для каждого понятия:
  - дай короткое и простое объяснение
  - если используется термин. которого не знает новичок. поясни его дополнительной фразой.

5.4. Содержание темы по блокам.
- раздел `## Содержание темы`
- разбей теорию на 3 или 4 блока. строго опираясь на структуру блоков из плана проведения в `notes.md`.
- для каждого блока создай подзаголовок вида.
  - `### Блок <номер>. <название блока>`

Для каждого блока выполни следующее.

5.4.1. Объяснение.
- дай подробное. но простое объяснение темы блока
- разбивай текст на небольшие абзацы
- избегай больших сплошных кусков текста

5.4.2. Примеры кода на Python.

5.4.3. Примеры из реальной жизни.
- для каждого блока приведи один или несколько примеров из реальной жизни, где используется описанная идея
- объясни связь между реальной ситуацией и кодом

## 6. Генерация теста по теоретическому файлу

Этот шаг выполняй только после того, как теоретический файл в `python2026` полностью сформирован.

6.1. Пути к тесту.
- репозиторий заданий. `python2026_tasks`
- папка урока.
  - `python2026_tasks/lessons/lesson <номер> (<дата>). <Название>/`
- в этой папке создавай или обновляй файл.
  - `test <номер урока>.json`

6.2. Если нужных папок в `python2026_tasks` нет.
- создай папки по пути.
  - `lessons`
  - `lessons/lesson <номер> (<дата>). <Название>/`

6.3. Формат файла теста.
- файл `test <номер урока>.json` должен содержать JSON массив из 10 объектов
- каждый объект вопроса имеет вид.
  ```json
  {
    "question": "Текст вопроса",
    "options": [
      "Вариант A",
      "Вариант B",
      "Вариант C",
      "Вариант D",
      "Вариант E",
      "Не знаю ответа"
    ],
    "scores": [0, 0, 1, 0, 0, 0.2]
  }
  ```
- поле `correct_index` не используй.
- массив `scores`.
  - содержит только одно значение с максимальным баллом. для правильного варианта
  - остальные основные варианты дают 0
  - вариант `"Не знаю ответа"` всегда шестой и дает частичный балл, например `0.2`.

6.4. Количество вопросов.
- ровно 10 вопросов
- если получается меньше 10 вопросов. дополни список вопросами по теории до 10
- не добавляй больше 10 вопросов.

6.5. Связь вопросов с теорией.
- каждый вопрос должен основываться только на теоретическом файле, который ты сгенерировал в `python2026`
- для ответа студенту не нужны другие источники или материалы
- вопросы должны покрывать все 3-4 блока теории.
  - каждый блок дает не менее 2 вопросов
- формулируй вопросы так, чтобы они проверяли понимание ключевых идей, а не запоминание формул или мелких деталей.

6.6. Порядок вопросов и вариантов.
- при генерации не перемешивай порядок вопросов
- при генерации не перемешивай порядок вариантов ответов
- пиши формулировки так, чтобы в будущем можно было безопасно перемешать как порядок вопросов, так и порядок вариантов.

6.7. Язык вопросов и ответов.
- используй простой язык
- избегай хитрых формулировок и подвохов
- если в вопросе используется термин, который может быть неочевиден новичку. коротко напомни его значение простыми словами.
